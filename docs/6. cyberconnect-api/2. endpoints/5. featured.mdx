The featured query returns a list of featured addresses by CyberConnect with their featured reason. It is a part of our [recommendation](/concepts/recommend/) system.

## Definition

The definition of "featured" query is:

```graphql
featured (fromAddr: String, network: Network) [Popular]!
```

For input params:

| Field      | Type    | Description                                                       |
| ---------- | ------- | ----------------------------------------------------------------- |
| `fromAddr` | String  | The address that you want to get popular for                      |
| `network`  | Network | The blockchain network for the queried address. Default is `ETH`. |

There are 6 fields for each object:

| Field                  | Type    | Description                                       |
| ---------------------- | ------- | ------------------------------------------------- |
| `address`              | String  | The string of featured address                    |
| `domain`               | String  | The string of featured address' domain name       |
| `avatar`               | String  | The URL string of featured address' avatar        |
| `recommendationReason` | String  | The reason why we featured this address           |
| `followerCount`        | Int     | The number of featured address' followers         |
| `isFollowing`          | Boolean | If the fromAddr is following the featured address |

## Example

This is an example of a query to fetch `featured` data that you can also test out in the [Playground](/cyberconnect-api/playground/):

```javascript
query QueryFeatured {
    featured(
        fromAddr: '0x7c04786f04c522ca664bb8b6804e0d182eec505f'
        network: ETH
    ) {
        address
        domain
        avatar
        recommendationReason
        followerCount
        isFollowing
    }
}
```
